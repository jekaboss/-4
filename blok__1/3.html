Архітектура «Чертогів Фрілансера» 4. Файли та папки

Розархівувавши збірку, в корені ви виявите такі файли та папки:

1. Папка src - містить похідні файли та папки проекту. Саме тут працює розробник

2. Файл template.config.js, містить користувацькі налаштування ЧФ4

3. Файл README.txt - важлива коротка інформація щодо використання ЧФ4

4. Файл cover.jpg - обкладинка ЧФ4

5. Файл package.json, містить команди запуску збирачів, інформацію про встановлені плагіни та їх версії. А також іншу інформацію про автора та версію збірки. Не рекомендується редагувати без відповідних знаннь

6. Файл docker-compose.yml, містись інструкції щодо встановлення та запуску серверу в режимі fullstack для роботи з CMS WordPress. Не рекомендується редагувати без відповідних знаннь

7. Папка template_modules містить функціонал збірки. Не рекомендується редагувати

8. Файл vite.config.js, містить інструкції для роботи системи VITE. Не рекомендується редагувати

9. При першому запуску будь-якого режиму, з'явиться папка .vscode, яка буде містити налаштування редактора та сніпети (якщо це дозволено в налаштуваннях ЧФ4)

10. При першому запуску команди npm run zip, з'явиться папка zip, яка буде містити zip-архів кінцевих файлів проєкту

11. При першому запуску режима fullstack (npm run wp, npm run wpbuild), з'явиться папка backend, яка містить папки wordpress та database. Не рекомендується редагувати без відповідних знаннь

12. Після інсталяції також автоматично з'являться папка node_modules та файл package-lock.json. Не редагувати

Вміст папки src:

1. Файл index.html - індексна (головна) сторінка проєкту що розробляється

2. Папка assets - використовується для зберігання медіа-файлів проєкту - зображеннь, шрифтів, іконок тощо

3. Папка styles - містить загальні файли стілів проєкту

4. Папка js - містить загальні файли скриптів проєкту, а також, папки та файли бібліотек та фреймворків

5. Папка pug - містить файли для роботи з препроцесором PUG

6. Папка php - містить PHP-компоненти та є головною папкою PHP-серверу якщо таке налаштування увімкнене

7. Папка files (порожня) - всі файли цієї папки будуть скопійовані в папку з результатом (dist/files)

8. Папка components - містить усі HTML/CSS/SCSS/JS компоненти ЧФ4, а також тему для CMS WordPress, шаблони HTML-сторінок та їх головний контент

Вміст папки src/components:

Містить усі HTML/CSS/SCSS/JS компоненти ЧФ4, а також тему для CMS WordPress, шаблони HTML-сторінок та їх головний контент

1. Папка custom (порожня) - тут можна створювати власні компоненти

2. Папка wordpress - містить тему для CMS WordPress, плагін та допоміжні файли. Використовується для розробки інтерфейся під CMS WordPress в режимі fullstack (npm run wp, npm run wpbuild)

2.1 fls-theme - тема для CMS WordPress

2.2 fls-plugin - власний плагін для CMS WordPress

2.3 JS-файли - допоміжні файли, не редагувати, не видаляти

3. Папка templates - містить шаблони HTML-сторінок, можна додавати свої

3.1 Папка main - містить HTML/SCSS/JS файли основного HTML-шаблону проєкту

3.2 Папка projectpage - містить HTML-шаблон сторінки проєкту

4. Папка pages - містить HTML/SCSS/JS файли основного контенту сорінок проєкту

4.1 Папка home - містить HTML/SCSS/JS файли основного контенту інтексної (головної) сторінки проєкту

5. Папка effects - містить папки та HTML/SCSS/JS файли компонентів які додають в проєкт різноманітні ефекти. Можна додавати свої

6. Папка forms - містить папки та HTML/SCSS/JS файли компонентів які додають в проєкт різноманітний функціонал для роботи з формами та їх елементами. Можна додавати свої

7. Папка layout - містить папки та HTML/SCSS/JS файли компонентів які додають в проєкт різноманітні частини інтерфейсу. Можна додавати свої

Додатково, папки компонентів можуть містити папки та файли плагінів конкретного компоненту, наприклад src/components/layout/header/plugins/scroll

Вміст папки src/assets:

1. Папка fonts (порожня) - використовується для конвертації та додавання локальних шрифтів в проєкт.

2. Папка img (порожня) - використовується для зберігання та використання зображень

3. Папка svgicons (порожня) - використовується для конвертації та додавання в проєкт іконкового шрифту та/або спрайту

Вміст папки src/styles:

Файл style.scss - головний файл стилів, який підключається до усіх HTML-сторінок.

Файл settings.scss - файл налаштуваннь загальних значень стилів проєкту

Папка fonts - містить автоматично заповнювані, при запуску будь-якого режиму, CSS-файли стилів підключення локальних та іконкового шрифтів

Папка includes - містить загальні CSS/SCSS-файли стілів

1. Файл index.scss - автоматично підключається до кожного SCSS-файлу проєкту та містить підключення налаштувань, SCSS міксінів та шаблонів. Можна підключати свої

2. Файл extends.scss - містить набір SCSS-шаблонів. Можна додавати свої

3. Файл mixins.scss - містить набір SCSS-міксінів. Можна додавати свої

4. Файл spritemap.scss - містить SCSS-міксін для роботи з SVG-спрайтом. Заповлюється автоматично

1. Папка libs - містить автоматично підключаємі до HTML-сторінок CSS-файли

1.1 Файл reset.css - автоматично підключається до кожної HTML-сторінки, якщо не використовується інша CSS-бібліотека/фреймворк

1.2 Файл tailwind.css - автоматично підключається до кожної HTML-сторінки, якщо використання вказано в налаштуваннях ЧФ4

Вміст папки src/js:

1. Файл app.js(порожній) - основний файл, загальних для усього проєкту, скриптів. Підключається до кожної HTML-сторінки

2. Папка common - містить файли загальних скриптів 2.1 Файл functions.js - містить корисні функції, які використовуються в компонентах ЧФ4

3. Папка react - містить стартові файли та папки для роботи з React

4. Папка vue - містить стартові файли та папки для роботи з Vue
